Hey Cinematic Composers

Many of you responded to my recent emails with a specific challenge: talk about specific MIDI programming techniques instrument by instrument.

Today I'm sharing my complete guide to programming the main orchestral instrument realistically, based on techniques I've developed over years of professional film scoring. This knowledge took me years to accumulate, but will transform your mockups almost immediately.

THE SECRET TO REALISTIC PROGRAMMING

Before diving into specific instruments, remember this fundamental truth: Great mockups don't try to perfectly recreate a real orchestra; they create a convincing illusion of one.

This distinction is crucial. We need to understand how real instruments work, but then adapt that knowledge specifically for the sample libraries we're using.

Let's explore each orchestral section in detail:

STRINGS: CREATING EMOTIONAL REALISM

String samples often sound the most "fake" when programmed poorly. Here's how to make them expressive and realistic:

1. Violin Programming Essentials

Articulation Selection Logic:

  • Legato for connected melodic lines (60-70% of usage)

  • Spiccato for passages below 120 BPM, staccato for faster passages

  • Measured tremolo for repeated notes

  • Portamento sparingly (only 5-10% of legato transitions)

  • Con sordino for gentle background textures

CC Control Strategy:

  • CC1 should never remain static - create gentle waves (Â±5-10%) even during "sustained" notes

  • Emphasize phrase beginnings with slightly higher CC1/CC11

  • For emotional peaks, push CC1 to maximum at the climax point, not before

  • Add subtle CC11 dips (10-15%) between slurred notes

Timing Adjustments:

  • First violins often lead slightly (5-8ms ahead of beat)

  • Create slight timing differences between Violin I and II (5-10ms)

  • Extend notes slightly beyond notated length for legato passages

  • Shorten notes to 85-90% length for detached passages

2. Viola and Cello Realism Techniques

Register Awareness:

  • Lower positions sound more resonant (more CC1, less CC11)

  • Upper registers need more careful dynamic control

  • Middle register often needs subtle EQ enhancement (slight boost around 400-600Hz)

Performance Techniques:

  • Program slightly less vibrato than violins

  • Allow longer transition times between notes

  • Create slightly more pronounced dynamic swells in sustained passages

  • For exposed viola/cello lines, add more CC11 movement than you think necessary

3. String Ensemble Coherence

Creating Unified Sections:

  • Maintain consistent dynamic contours across entire sections

  • Subtle timing variations between desks (5-10ms maximum)

  • Program "desk by desk" rather than note by note

  • Velocity variation of Â±5-7% within sections

Layering Strategies:

  • For powerful emotional passages, layer legato with subtle marcato (15-20% volume)

  • For textural passages, consider layering con sordino under normal strings (10-30% volume)

  • For aggressive passages, layer spiccato under sustains for enhanced attacks

Pro Tip for Strings: Real string players connect phrases with minimal breaks. Extend final notes of phrases slightly beyond their written length and overlap with the first note of the next phrase by 60-80ms.

BRASS: POWER AND PRECISION

Brass requires a completely different approach than strings to sound convincing:

1. French Horn Programming

Dynamic Mapping:

  • Restrict the dynamic range based on register (lower notes = more restricted range)

  • Create "blooming" notes by starting at mp and swelling to target dynamic

  • Allow 80-120ms for notes to reach full dynamic level

  • Add subtle dynamic "arc" to sustained notes

Performance Nuances:

  • Add slight pitch bend at note beginnings for realism

  • Program "breathmarks" between phrases (80-150ms gaps)

  • Reduce note lengths to 85-90% of written value

  • Use subtle note overlaps (40-60ms) within slurred passages

2. Trumpet Programming Techniques

Attack Characteristics:

  • Faster attacks than horns (30-60ms to reach target dynamic)

  • More aggressive CC1 curves

  • More pronounced dynamic contrast

  • Clear separation between notes in non-slurred passages

Register Treatment:

  • Lower register needs gentler attacks

  • Upper register requires careful dynamic control

  • Extreme upper register should be used sparingly and with appropriate crescendos

3. Low Brass Realism

Trombone Specifics:

  • More pronounced "attack bump" on CC1 at note beginnings

  • Slower glissandi between slurred notes

  • Shorter note lengths than written (except for fortissimo passages)

  • Low register needs room to develop (120-180ms attack time)

Tuba Specifics:

  • Significantly restricted dynamic range

  • Much slower attacks (150-200ms)

  • Allow substantial pre-delay when aligning with other brass

  • Add subtle pitch instability for realism

4. Brass Ensemble Techniques

Section Cohesion:

  • Align attacks precisely (within 5-10ms)

  • Create unified dynamic shapes

  • Add slight timing variation between different brass instruments

  • Unison passages need subtle detuning (Â±3-5 cents)

Dynamics Management:

  • Never exceed 90% CC1 in mid-registers except for fortissimo passages

  • Create dynamics "ceiling" appropriate to the register

  • Apply consistent dynamic limits across all brass for coherence

Pro Tip for Brass: Most sampled brass instruments sound most realistic in their mid-dynamic ranges. Even for fortissimo passages, try keeping CC1 around 80-90% and using CC11 to increase perceived volume instead of pushing to maximum CC1.

WOODWINDS: AGILITY AND EXPRESSION

Woodwinds require the most detailed programming of all orchestral sections:

1. Flute Programming Essentials

Breathing Strategy:

  • Program distinct breathing points (100-150ms gaps)

  • Never sustain notes at consistent dynamics

  • Create dynamic "arcs" within each note

  • Long notes should have subtle internal movement (vibrato and dynamics)

Performance Details:

  • Faster passages need sharper attacks

  • Dynamic range should vary by register (less extreme in low register)

  • Add slight pitch variation for tuning realism (Â±5 cents)

  • Use CC1 for dynamics and CC21 for vibrato depth

2. Clarinet and Oboe Techniques

Register Treatment:

  • Different dynamics for different registers (especially clarinet)

  • Careful handling of "break" notes on clarinet (around F#4-G4)

  • More restricted dynamic range for oboe

  • Slower attacks for lower registers

Articulation Details:

  • Shorter staccato notes than other woodwinds

  • More pronounced dynamic contrast between slurred notes

  • Less vibrato than flutes, especially for clarinet

  • Longer transition times between legato notes

3. Bassoon Programming

Special Considerations:

  • Limited dynamic range in low register

  • Add "grunt" at beginnings of low notes (slight CC1 bump)

  • Slower attacks than other woodwinds

  • Less vibrato overall

Expression Techniques:

  • More pronounced dynamic swells than other woodwinds

  • Gentle pitch variation for realism

  • Allow more time for registral transitions

  • Careful handling of staccato articulations (slightly longer than other woodwinds)

4. Woodwind Ensemble Programming

Creating Natural Sections:

  • Avoid perfect unison - offset timing by 5-15ms between players

  • Create independent dynamic shapes for each player

  • Vary vibrato rates between instruments

  • Allow slight tuning differences (Â±3-7 cents)

Special Effects:

  • For flutter tonguing, use dedicated samples or add subtle LFO

  • For falls, program custom pitch bends rather than using generic samples

  • For runs, use dedicated run patches or create custom legato runs

Pro Tip for Woodwinds: Most sample libraries don't capture the "instability" of real woodwinds. Add subtle CC and pitch variation throughout even seemingly "static" notes to create realism.

PERCUSSION: IMPACT AND TIMING

Percussion requires a completely different mindset than melodic instruments:

1. Timpani Programming

Dynamic Mapping:

  • Soft hits (pp-mp): velocity 1-60

  • Medium hits (mp-f): velocity 61-100

  • Loud hits (f-ff): velocity 101-127

Performance Techniques:

  • Rolls need custom programming (don't rely on roll patches)

  • Create crescendo rolls with increasing rate AND velocity

  • Program discrete hits for rolls rather than using sustain samples

  • Add subtle timing variation between hits (5-10ms)

2. Orchestral Percussion Techniques

Individual Instrument Approaches:

Snare Drum:

  • Vary velocities within patterns (Â±10-15%)

  • Use different samples for accented notes

  • Program realistic crescendos for rolls

  • Add ghost notes at very low velocities (10-25) for realism

Cymbals:

  • Use appropriate samples based on dynamic level

  • Allow full decay for crashes

  • For choked cymbals, program exact choke points

  • Layer multiple cymbal types for complex sounds

Bass Drum:

  • Allow appropriate decay based on tempo

  • Vary velocities for repeated patterns

  • Use different articulations based on musical context

  • Program subtle release triggers where available

3. Creating Realistic Percussion Ensembles

Ensemble Coordination:

  • Add slight timing variations between players (5-15ms)

  • Vary velocities between different percussion instruments

  • Create dynamic hierarchies (primary rhythm vs. supporting elements)

  • Program accent patterns that follow musical phrasing

Layering Strategy:

  • Layer similar instruments for complex sounds

  • Combine close and room mics for depth

  • Use different sample libraries for complementary tones

  • Create custom percussion ensembles for specific needs

Pro Tip for Percussion: Most percussion samples sound best when not quantized to the grid. Add 5-15ms of humanization to all percussion parts, but maintain consistent internal timing within each percussion instrument's pattern.

HARP AND PIANO: DETAIL AND NUANCE

These instruments require specific approaches:

1. Harp Programming

Realistic Arpeggios:

  • Vary velocity slightly through arpeggios (typically descending velocity)

  • Add timing variation between notes (increasing spacing for lower notes)

  • Use proper damping between phrases

  • Program realistic glissandi with appropriate timing

Performance Details:

  • Allow notes to ring except where damping is indicated

  • Harmonics need lower velocities than regular notes

  • Add slight timing offset for very low and very high notes

  • Use appropriate velocity ranges (pp-mp: 30-70, mf-f: 71-95, ff: 96-115)

2. Piano in Orchestral Context

Integration Techniques:

  • Balance with appropriate reverb for orchestral context

  • Use close mics with orchestral reverb sends

  • Program pedaling carefully based on musical context

  • Allow more dynamic contrast than other orchestral elements

Performance Nuances:

  • Add subtle timing variation between hands

  • Use proper velocity curves based on musical style

  • Program partial pedaling for delicate passages

  • Add subtle dynamics movement for sustained chords

BRINGING IT ALL TOGETHER: ENSEMBLE COHESION

The final step in realistic programming is ensuring all instruments work together:

1. Prioritization Hierarchy Not all instruments should be equally prominent:

  • Establish clear foreground, middleground, and background elements

  • Program dynamic ranges appropriate to each layer

  • Create more expressive controller data for featured instruments

  • Allow background elements to be more consistent and subtle

2. Unified Phrasing Make instruments feel like they're playing together:

  • Create compatible dynamic shapes across sections

  • Align breathing/phrasing points

  • Program coordinated crescendi and diminuendi

  • Maintain consistent articulation approaches within sections

3. Balance MIDI CC Data Across Sections

  • Brass typically needs more dramatic CC curves than strings

  • Woodwinds need more internal movement than brass

  • Strings require more subtle continuous variation

  • Percussion dynamics should align with overall dynamic contours

CONCLUSION: THE PROGRAMMING MINDSET

After years of creating mockups, I've found that the most important skill is thinking like both a musician and a programmer simultaneously.

Great MIDI programming isn't about perfect technical execution - it's about understanding the musical purpose behind every note.

Ask yourself:

  • What is the emotional intent of this passage?

  • How would a real player approach this phrase?

  • What subtle details would make this more convincing?

  • How does this instrument interact with others around it?

These questionsâmore than any specific techniqueâwill guide your programming decisions toward truly realistic results.
